For when you don't know what to build yet.

## When to use

- You have frustrations but no clear problem
- You have multiple ideas and can't choose
- You're not sure if something is worth building

## What it does

1. **Diverge** — "What's bugging you?" Generate breadth. Push for more ideas until you run dry.
2. **Cluster** — Group related ideas. Present the clusters back.
3. **Challenge** — Not all pain is worth solving. Kill weak clusters.
4. **Converge** — Narrow to 1-3 candidates. State each as: problem, why it matters, what "solved" looks like.
5. **Choose** — Help pick one. "Which one, if you solved it, would make the other two easier?"
6. **Hand off** — Write the problem to `specs/TODO.md` and suggest `/upfront:feature`.

## Output

No spec, no file. The output is clarity — plus a one-line note in `specs/TODO.md` so it survives context resets.

## Next step

`/upfront:feature` to define it properly.
