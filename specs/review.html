Commands

/feature changed

~/.claude/commands/feature.md

Define a feature through intent, behavioral spec, design, and implementation design.
  • Phase 1: Intent (5 forcing-function questions with pushback)
  • Phase 2: Behavioral Spec (5-level funnel)
    • Stories, Mechanism, States
    • NEW: Level 4 — Concurrency and shared state (races, read-modify-write, partial completion, retry/idempotency, lock ordering, cache invalidation)
    • Error cases and edges
  • Phase 3: Design Conversation (codebase research, options, tradeoffs)
  • Phase 4: Implementation Design (placement, architecture, blind spots)
  • Thinking records at every phase transition

/plan changed

~/.claude/commands/plan.md

Break a spec into ~400 LOC implementation phases by researching the codebase.
  • Read spec, research codebase
  • NEW: 3-level architectural deep-dive
    • Level 1: System architecture (components, communication, invariants)
    • Level 2: Subsystem architecture (modules, boundaries, data models)
    • Level 3: Design patterns and connections (interfaces, edge behaviors, error propagation)
    • Each level confirmed by user before proceeding
    • Challenges assumptions, pushes for well-understood behaviors
  • NEW: Architecture persisted to specs/ARCHITECTURE.md — shared across features, reviewed every /plan run
  • Propose phases with verification criteria

/build new

~/.claude/commands/build.md

Execute implementation plan phase by phase with strict TDD and post-phase review.
  • Fresh sub-agent per phase (RALPH pattern — prevents context degradation)
  • Strict TDD enforced — Red/Green/Refactor cycle required. Work rejected if TDD skipped.
  • Post-phase code review — separate review agent checks correctness, architecture, spec compliance
  • MUST FIX / SHOULD FIX / NOTE — MUST FIX blocks proceeding, SHOULD FIX presented to user
  • Progress file tracks learnings between phases
  • Sequential commits per phase (easy to revert individually)
